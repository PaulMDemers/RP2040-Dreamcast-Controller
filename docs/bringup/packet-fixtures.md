# Packet Fixtures

These host-order packet words are generated by `make sim`. They are useful as early logic-analyzer and transport test references.

The PIO TX buffer uses wire-order words plus a final CRC word whose high byte is shifted first. For the get-condition request below, the PIO-facing data words are:

```text
bit_count: 72
0x01002009
0x01000000
0x29000000
```

## Device Info Request

```text
0x01200000
checksum: 0x21
```

Meaning:

- command: `0x01` device info request
- recipient: `0x20` main peripheral on port A
- sender: `0x00` Dreamcast host on port A
- payload words: `0`

## Get Condition Request

```text
0x09200001
0x00000001
checksum: 0x29
```

Meaning:

- command: `0x09` get condition
- recipient: `0x20` main peripheral on port A
- sender: `0x00` Dreamcast host on port A
- payload words: `1`
- payload: `0x00000001` controller function

## Example Get Condition Response

State:

- A pressed
- Start pressed
- right trigger: `200`
- left stick X/Y: `180`, `80`

```text
0x08002003
0x00000001
0xF3FFC800
0xB4508080
checksum: 0x0A
```

Condition bytes in little-endian memory order:

```text
00 C8 FF F3 80 80 50 B4
```

Decoded:

- left trigger: `0`
- right trigger: `200`
- high button byte: `0xFF`
- low button byte: `0xF3`, so A and Start are active-low pressed
- right analog Y/X: `128`, `128`
- left analog Y/X: `80`, `180`
